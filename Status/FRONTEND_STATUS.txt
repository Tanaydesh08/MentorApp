Frontend currently implemented
1. Routing and pages
- `/`
- `/login`
- `/register`
- `/dashboard`
- `/session/[sessionId]`

2. Auth flow
- register and login UI exist
- JWT token is stored in `localStorage`
- protected frontend pages redirect to `/login` when token is missing
- helper logic is centralized in `src/lib/auth.ts`

3. Dashboard
- `Create Session` generates a 6-character session id and routes to `/session/[sessionId]`
- `Join Session` opens an existing session id in the same tab
- dashboard loads `GET /api/users` for authenticated users
- signed-in email and role are shown from the decoded token

4. Session page layout
- layout follows the `mentorApp.drawio.png` reference closely
- left side: video/call area
- right top: Monaco editor
- right bottom: chat

5. Realtime socket layer
- frontend connects to `ws://localhost:8080/ws/session`
- query params sent:
  - `sessionId`
  - `token`
  - `clientId`
- socket handles:
  - presence
  - system messages
  - chat
  - editor sync
  - WebRTC signaling

6. Chat
- chat UI exists in the session page
- sent messages are rendered immediately in the sender UI
- same-tab websocket echoes are ignored to prevent duplicates

7. Shared editor
- Monaco editor integrated
- JavaScript and Python selection available
- code sync is room-based over websocket
- sync model is simple last-write-wins broadcast

8. Video/audio/screen share UI
- local media starts with `getUserMedia`
- mic toggle exists
- camera toggle exists
- screen sharing exists with `getDisplayMedia`
- local preview and remote video area are both wired

9. WebRTC signaling
- `RTCPeerConnection` is set up in the session page
- signaling messages exchanged through websocket:
  - description offer/answer
  - ICE candidates
- negotiation ordering now uses a per-tab `clientId`, not just email

10. Backend support already in place for frontend
- websocket endpoint exists
- websocket handshake validates token and session id
- websocket presence and relay logic exists
- `/api/users` is now authenticated, not mentor-only
