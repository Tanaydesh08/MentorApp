Current database implementation
1. Connection and ORM
- `application.properties` points Spring Boot to Oracle at `jdbc:oracle:thin:@localhost:1521:XE`.
- JPA is enabled and `spring.jpa.hibernate.ddl-auto=update` is active.
- This means Hibernate is handling schema updates at runtime instead of a migration tool.

2. Actual entity/table coverage
- Only `entity/User.java` is mapped right now.
- The current backend logic expects a `users` table with:
  - `id`
  - `email`
  - `password`
  - `role`
- `UserRepository` only supports user lookups, mainly `findByEmail(...)` plus standard JPA methods.

3. What the database is used for today
- Registration inserts new users.
- Passwords are BCrypt-hashed before save.
- Login loads a user by email and checks the stored hash.
- `GET /api/users` reads all stored users.

4. SQL script reality
- `mentor-db.sql` is a manual scratchpad file, not a migration system.
- It contains ad hoc selects, inserts, deletes, and a manual unique-email constraint statement.
- The entity model itself does not declare `email` as unique in code.

What is missing in SQL
1. Session data model
- no `sessions` table
- no session owner/creator field
- no start time, end time, or session status
- no persistent session code storage

2. Invitation data model
- no `session_invitations` table
- no sender/receiver references
- no pending/accepted/rejected state
- no invitation timestamps

3. Message persistence
- no `messages` table
- no sender/session foreign keys
- no saved chat history

4. Editor persistence
- no saved code snapshots per room
- no stored selected language per session
- no audit trail of editor changes

5. Relationship and constraint design
- no explicit foreign-key mapping between sessions and users
- no indexing strategy for common lookups
- no standard audit columns like `created_at` or `updated_at`
- no role constraint/check in the entity model

6. Migration discipline
- no Flyway or Liquibase setup
- no versioned SQL history
- schema changes currently depend on runtime Hibernate behavior

Honest SQL maturity summary
- Oracle is connected and used.
- User authentication storage exists.
- The product data model for sessions, invites, messages, history, and reporting is still not implemented.
- SQL is one of the least complete parts of the project today.
